# ADR 0003：任务默认只读

状态：已接受
日期：2026-05-13

## 背景

z-fleet 不是远程控制工具、EDR，也不是无约束命令执行平台。项目早期价值来自资产可见、状态可知、变更可审计、任务可控和更新可信。

## 决策

任务默认使用 `readonly` 能力等级。写入能力必须通过策略显式开启，高风险写入和 `shell` 执行不属于早期里程碑范围。

## 备选方案

- 早期直接开放 `shell` 或任意写入任务：灵活性更高，但与项目安全模型冲突。
- 不提供任务系统：初期更保守，但 v0.2 需要受控只读任务和可审计结果回传。

## 影响

- 每种任务类型都必须定义输入、输出、状态流转、错误、所需权限、审计字段和测试。
- 非 `readonly` 能力在实现前必须先完成威胁建模。
- 系统可以逐步演进到管理能力，但不应把无约束执行常态化。
