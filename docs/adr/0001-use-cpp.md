# ADR 0001：Agent 与 Server 使用 C++

状态：已接受
日期：2026-05-13

## 背景

z-fleet 面向低资源占用、跨平台部署、长期运行进程，以及 Agent 与 Server 的可控打包发布。项目还需要直接访问平台 API 以完成资产采集和服务管理。

## 决策

使用 C++20 作为 Agent、Server 和共享库的主要实现语言。

## 备选方案

- Go：部署简单、网络支持强，但不如当前确定的 C++ 方向以及原生平台抽象目标契合。
- Rust：安全模型强，但对当前代码库和团队工作流而言，初期复杂度更高。
- Python：迭代快，但不适合 Agent 对资源占用和部署形态的目标。

## 影响

- 项目可以在 Agent 与 Server 之间共享通用协议库、平台库和核心库。
- 依赖管理、构建脚本、打包和 CI 必须保持克制和规范。
- 内存安全需要依赖代码评审、测试、在可行处启用 sanitizer，以及明确的依赖边界。
