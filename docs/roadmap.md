# 路线图

状态：草案
最后更新：2026-05-24
关联里程碑：v0.1, v0.2, v0.3

## v0.1 最小闭环

目标：产生可运行、可验证的端到端闭环。

计划范围：

- Agent 启动后生成或读取本地身份；
- Agent 主动连接 Server；
- Server 接收注册和心跳；
- Agent 上报基础资产信息；
- Server 持久化设备、心跳和资产快照；
- 注册、心跳、资产上报和错误路径生成审计事件；
- 提供最小测试和本地运行说明。

暂不包含：

- Shell；
- 插件；
- 自更新；
- 复杂 UI；
- 高风险写入任务。

## v0.2 任务与审计模型

目标：建立受控任务能力。

计划范围：

- 任务模型、状态机和错误码；
- 只读任务下发和结果回传；
- 任务审计日志；
- 策略开关和任务能力分级；
- 协议版本和兼容性约束。

## v0.3 更新与跨平台验证

目标：提升可部署性和可靠性。

计划范围：

- 自更新签名、灰度、回滚和失败保护；
- Linux 与 Windows 最小真实设备验证流程；
- 包构建、发布和回滚文档；
- 配置迁移和 Agent / Server 兼容策略。
- Server 内置 Web 管理入口最小闭环：
  - Agent 安装页；
  - Agent 状态列表和资产详情；
  - Agent 安装包上传、校验、channel 发布和退役；
  - 注册 token 生成、过期、吊销和审计。

## 长期方向

- 受限插件系统；
- 更完整的资产模型；
- 更丰富的策略和审批流程；
- Web 管理面的权限分层、审批流、报表和更复杂灰度策略；
- 多设备巡检报告；
- AI 辅助审查、测试生成和运维报告集成。
